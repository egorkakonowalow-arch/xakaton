Ты — Senior Fullstack Developer (Backend focus). Твоя задача — написать ПОЛНОСТЬЮ РАБОЧИЙ асинхронный backend для платформы управления региональным отделением. 

СТЕК ИЗ REQUIREMENTS.TXT:
- FastAPI, SQLAlchemy 2.0 (Async), aiosqlite, Pydantic v2, Passlib (bcrypt), Alembic, python-multipart (для файлов).

АРХИТЕКТУРА (Обязательно):
Соблюдай строгую структуру: Endpoints -> Services -> Repositories -> Models.
В начале каждого файла добавь подробный docstring на русском языке: описание файла, его роль в системе и логика работы.

ДЕТАЛЬНОЕ ТЕХНИЧЕСКОЕ ЗАДАНИЕ ДЛЯ РЕАЛИЗАЦИИ:

1. СИСТЕМА АУТЕНТИФИКАЦИИ (Auth Service):
- Реализуй JWT-авторизацию (access/refresh токены).
- Роли: 'admin', 'head' (руководитель), 'employee' (сотрудник).
- Dependency injection get_current_user с проверкой ролей (scopes).

2. КОНСТРУКТОР ДИНАМИЧЕСКИХ ФОРМ (Admin Logic):
- Реализуй эндпоинты для создания шаблонов отчетов (`FormTemplate`) и полей к ним (`FormField`).
- Типы полей: текст, число, дата, файл.
- Логика: При создании отчета сотрудником, система должна валидировать входящий JSON на соответствие полям из шаблона.

3. ПЛАНИРОВАНИЕ И ЗАДАЧИ (Workflow Logic):
- Реализуй логику: 'head' создает план на месяц и привязывает к нему 'PlanItem' (мероприятия).
- Из 'PlanItem' руководитель может создать 'Task' для конкретного сотрудника.
- СТАТУСЫ: Задачи имеют статусы (new, in_progress, completed). 
- АВТОМАТИЗАЦИЯ: Если задача создана на основе мероприятия плана, то при успешной сдаче отчета (`Report`) по этому мероприятию, статус задачи должен автоматически меняться на 'completed'.

4. ПРАВА ДОСТУПА (Row Level Security):
- Админ: CRUD всех сущностей.
- Руководитель: видит отчеты, планы и задачи только тех районов (`districts`), которые к нему привязаны через user_districts.
- Сотрудник: видит только свои задачи и может создавать отчеты только по своему району.

5. РЕПОЗИТОРИИ (Database Layer):
- Напиши асинхронный BaseRepository с методами find_one, find_all, add, update, delete.
- Реализуй конкретные репозитории для каждой сущности, наследуясь от базового.

ЧТО НУЖНО СДЕЛАТЬ СЕЙЧАС (Full Implementation):
Напиши код для следующих модулей, полностью реализуя их логику:

Шаг 1: Настройка ядра (app/db/session.py, app/core/config.py, app/core/security.py).
Шаг 2: Модели SQLAlchemy (app/models/) — опиши все связи (relationship) между пользователями, районами, формами и задачами.
Шаг 3: Pydantic схемы (app/schemas/) — включая вложенные схемы для динамических форм.
Шаг 4: Репозитории и Сервисы (app/services/) — реализуй бизнес-логику проверки прав и автоматического закрытия задач.
Шаг 5: Роутеры (app/api/v1/) — создай эндпоинты для:
   - /auth (login, register)
   - /districts (admin only)
   - /tasks (assign, update status, get list)
   - /reports (submit report, get analytics)
   - /forms (template creation)

Пиши код асинхронно. Используй Annotated для зависимостей FastAPI. Убедись, что каждый файл содержит описание логики. Начни с моделей и базового репозитория.
