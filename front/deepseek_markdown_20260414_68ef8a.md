## 1. Цель
Инструмент для автоматизации управленческих процессов:  
- сбор отчетности  
- планирование деятельности  
- постановка и контроль задач  
- анализ эффективности  

## 2. Функциональные модули (каждый — рабочий)

### 2.1. Модуль сбора и хранения данных
- Форма создания новой формы (динамические поля: текст, число, дата, файл — файл имитировать, хранить только имя).  
- Список созданных форм.  
- Форма для заполнения любой созданной формы (разные пользователи).  
- Все ответы хранятся в localStorage, отображаются таблицей.  
- Поддержка загрузки файлов (псевдо: показывать имя файла, не отправлять на сервер).

### 2.2. Модуль планирования деятельности
- Конструктор планов: кнопка «Создать план», поля: название, тип (отделение/мероприятие/проект), сроки, ответственный.  
- Шаблоны планов: предустановленные (например, «Мероприятие» с полями цель, бюджет). Выбор шаблона при создании.  
- Редактирование планов.  
- История изменений (лог: дата, пользователь «Admin», что изменил).

### 2.3. Модуль постановки и контроля задач
- Создание задачи: название, срок, ответственный (выбор из предустановленных: Иванов, Петров, Сидорова), описание, группа (например, «Отдел продаж»).  
- Статусы: «в работе», «выполнено», «просрочено» (автоматически, если срок < сегодня и статус не выполнен).  
- Уведомления: при создании задачи — alert или toast; при дедлайне за 1 день — метка на задаче.  
- Отметка выполнения: кнопка «Выполнить» → статус меняется.

### 2.4. Модуль аналитики и отчётности (ОБЯЗАТЕЛЬНО диаграммы через JS/React)
- **Диаграмма 1** (столбчатая или круговая) по числовому полю из собранных данных (выбор поля через select, данные берутся из ответов форм).  
- **Диаграмма 2** (линейная или столбчатая) — график выполнения задач по дням (количество завершённых задач за период).  
- **Дашборд** с карточками: всего задач, выполнено, просрочено; всего планов; всего отправленных форм.  
- **Фильтры** на дашборде: по периоду (date picker) и по ответственному.  
- **Экспорт** текущего дашборда в CSV (кнопка).

### 2.5. Модуль управления пользователями
- Переключатель ролей в интерфейсе (без реальной авторизации) для демонстрации прав:  
  - **Администратор** — видит все модули и все данные.  
  - **Руководитель** — видит задачи и планы своего отдела (отдел жёстко привязать к пользователю: например, руководитель видит только задачи с группой «Отдел продаж»).  
  - **Исполнитель** — видит только свои задачи (по ответственному).  
- Личный кабинет: ФИО (захардкодить «Демо Пользователь»), роль, персональные задачи, планы, где он ответственный, его отправленные формы.

- Код должен быть разбит на компоненты (например: DataCollection, Planning, Tasks, Analytics, UserProfile).

- **Никаких лишних функций** — только строго по списку выше.
